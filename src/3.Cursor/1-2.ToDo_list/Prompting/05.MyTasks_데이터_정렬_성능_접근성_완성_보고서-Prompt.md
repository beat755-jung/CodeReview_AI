render()의 힌트 삽입 로직이 잘못되어 있고, `updateDashboard`가 매 렌더마다 격언을 바꿉니다. 드래그는 핸들에서 시작하도록 수정합니다.



dragover에서 `render()`를 호출하면 드래그가 끊깁니다. 클래스만 갱신하도록 수정하고, 중복 다이얼로그 취소 시 편집 UI를 다시 그리도록 합니다.



작업 반영했습니다. 구현 요약만 정리합니다.

## 1. 데이터 보내기 / 가져오기
- **JSON보내기**: `version`, `app`, `exportedAt`, `sortMode`, `tasks`(각 항목에 `sortIndex` 포함) 구조로 Blob 다운로드.
- **JSON 가져오기**: `.json` 읽기 → 크기 제한(5MB) → JSON 파싱 오류·빈 파일·형식 불일치·잘못된 `createdAt` 행 제외·배열/래퍼 객체 모두 허용.
- **백업 확인**: 가져오기 전 `<dialog>`로 **취소 / 백업 없이 가져오기 / 백업 후 가져오기** 선택. “백업 후”는 현재 목록을 타임스탬프 파일명으로 저장한 뒤 교체.
- `localStorage` 할당 실패(용량 등) 시 토스트로 안내.

## 2. 정렬
- **생성일 / 카테고리 / 완료 여부 / 수동(드래그)** 라디오 그룹, 선택값은 `myTasks:sortMode:v1`에 저장.
- **수동**: `sortIndex`로 순서 유지, `ensureSortIndices()`로 정규화.
- **DnD**: `전체` 필터 + 검색 비어 있을 때만, **⋮⋮** 핸들에서만 드래그(버튼이 아닌 `span`+`draggable`로 체크박스와 충돌 방지). `dragover`에서는 **전체 `render()`를 호출하지 않음** → 드래그가 끊기지 않도록 클래스만 갱신.
- **키보드**: 핸들 포커스 시 **Alt+↑/↓**로 한 칸 이동(스크린리더·키보드 대안).

## 3. 성능
- **저장 디바운스**(`DEBOUNCE_SAVE_MS`), **`beforeunload`에서 `flushSaveTasks`**.
- **검색 디바운스** + `requestAnimationFrame`으로 `scheduleRender`.
- 목록은 **`DocumentFragment`**로 한 번에 붙여 DOM 갱신 최소화.

## 4. 접근성
- 데이터/정렬 영역 `role="region"|radiogroup"`, `aria-labelledby`, 목록 `aria-label`, 항목 `aria-label`(완료·카테고리·본문·시간).
- 체크박스/삭제/추가 버튼에 의미 있는 **`aria-label`**.
- 중요 알림은 **`aria-live="assertive"`** 영역 + `role="alert"`.
- 다이얼로그 닫은 뒤 **`focusReturnEl`**로 포커스 복귀.

## 5. 기타
- **중복**: 추가·수정 시 문구 정규화 후 중복이면 확인 다이얼로그.
- **Undo**: 단일 삭제 후 하단 바에서 **실행 취소**(타이머 만료·가져오기·완료 일괄 삭제 시 무효).
- **격언**: 날짜+`sessionStorage`로 같은 날·같은 탭에서는 고정, 다른 날이면 새 랜덤.
- **응원**: 완료율 구간별 메시지를 대시보드에 반영.

파일: `index.html`(UI·다이얼로그·힌트), `script.js`(주석·로직), `style.css`(툴바·DnD·Undo·토스트 등).

원하면 다음도 이어서 할 수 있습니다: 가져오기 시 “병합” 모드, Undo 여러 단계, 가상 스크롤(수천 건).

프로젝트 구조와 기존 UI 코드를 확인한 뒤 `web_version` 폴더에 데스크톱용 레이아웃을 구성합니다.



script.js에서 DOM 선택자와 레이아웃 의존성 확인 중.



`web_version` 폴더를 만들고 `script.js`와 `style.css`를 복사한 뒤, HTML 구조와 데스크톱용 CSS를 적용합니다.



`web_version/index.html` 작성 및 `web_version/style.css`에 데스크톱 레이아웃 미디어쿼리 추가.



모바일(960px 이하)에서 헤더가 먼저 보이도록 `flex`와 `order`로 순서를 조정합니다.



모바일에서 자연스러운 읽기 순서(헤더 → 할 일 영역 → 요약)를 위해 HTML을 `app__main`을 먼저 두고, 그리드에서 사이드바를 왼쪽 열에 고정합니다.



CSS에 그리드 열 배치(`main` → 오른쪽, `sidebar` → 왼쪽)를 추가하고, 모바일 미디어쿼리에서 불필요한 `order` 규칙을 제거합니다.



작업을 반영했습니다. 요약만 정리합니다.

## `web_version` 폴더

| 파일 | 설명 |
|------|------|
| `web_version/index.html` | 데스크톱용 2열 구조(메인 + 좌측 패널). 제목은 `My Tasks (웹)`으로 구분 |
| `web_version/style.css` | 루트 `style.css` 복사 + **961px 이상** 데스크톱 레이아웃 규칙 추가 |
| `web_version/script.js` | 루트와 동일(로직·저장소 키 동일 → 같은 브라우저에서 데이터 공유) |

## 레이아웃 (960px 초과)

- **최대 너비 약 1680px**, 가로로 넓게 사용
- **왼쪽 사이드바(`aside`)**: 대시보드, JSON 보내기/가져오기, 정렬, 카테고리 필터 — `position: sticky`로 스크롤 시에도 같은 화면에서 요약·필터 유지
- **오른쪽 메인**: 헤더(제목·테마), 검색·배지, 입력 폼, 목록 — 할 일 리스트는 `flex`로 세로 공간을 더 쓰도록 **최소 높이**를 둠
- DOM 순서는 **메인 → 사이드바**라서, 좁은 화면에서는 기존처럼 **헤더·입력·목록이 먼저** 나오고, 아래에 요약·필터가 이어집니다.
- 그리드에서는 `grid-column`으로 **시각적으로만** 사이드바를 왼쪽에 배치했습니다.

## 사용 방법

브라우저에서 `web_version/index.html`을 열면 됩니다(루트 `index.html`과 같은 `localStorage` 키를 쓰므로, 같은 출처에서 열면 할 일 데이터가 그대로 보입니다).

추가로 원하시면 **사이드바 폭**, **브레이크포인트(961px)**, **울트라와이드에서 목록 최대 너비** 같은 값도 맞춰 드리겠습니다.

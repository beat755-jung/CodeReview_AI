## 1. 개요

할 일 **추가 시** 제목 문자열에 포함된 **키워드**를 검사하여 `work` / `personal` / `study` 중 하나로 **카테고리를 자동 선택**하는 기능을 추가하였다. 사용자는 **「키워드로 자동 분류」** 체크박스로 이 동작을 끄거나 켤 수 있으며, 키워드와 맞지 않을 때는 기존과 같이 **카테고리 드롭다운** 값이 사용된다. 데이터 스키마(`category` 필드)는 변경하지 않았다.

---

## 2. 사용자 인터페이스

| 요소 | 설명 |
|------|------|
| 체크박스 | `#auto-category` — 라벨 문구 「키워드로 자동 분류」, HTML 기본값 `checked` |
| 힌트 영역 | `#task-category-hint` — 자동 분류가 **켜져 있을 때만** 표시, `aria-live="polite"` |
| 레이아웃 | `#task-form` 하단 `.task-form__options` — 체크박스와 힌트를 한 줄(줄바꿈 가능)로 배치 |
| 스타일 | `.task-form__options`, `.task-form__auto-label`, `.task-form__category-hint` — 구분선·톤 다운 힌트 |

### 2.1 힌트 문구 동작

- 입력이 비어 있음: 「할 일을 입력하면 키워드 추천이 표시됩니다.」
- 매칭 성공: 「추천: {업무\|개인\|공부}」
- 매칭 없음: 「키워드 없음 · {선택된 드롭다운 라벨} 카테고리로 추가됩니다.」
- 자동 분류 **끔**: 힌트 비움 + `hidden` 처리

입력·드롭다운 변경 시 힌트는 **디바운스(200ms)** 후 갱신하여 입력 중 과도한 갱신을 줄인다.

## 3. 분류 알고리즘

### 3.1 키워드 테이블

- 상수 객체 **`CATEGORY_KEYWORDS`**: 카테고리 키별 문자열 배열(한·영 혼합).
- 빌드 시 각 키워드를 `trim` 후 **소문자**로 정규화하여 **`KEYWORD_ENTRIES`** 배열을 만든다.

### 3.2 매칭 순서

1. 할 일 제목을 `String(...).toLowerCase()` 한 문자열(바늘)로 둔다.
2. `KEYWORD_ENTRIES`를 순서대로 검사하며, 바늘에 **`indexOf(키워드) !== -1`** 이면 해당 항목의 카테고리를 반환한다.
3. 정렬 규칙: **키워드 길이 내림차순** 우선(긴 구문이 짧은 단어에 가려지지 않도록), 동일 길이면 **업무 → 개인 → 공부** 순(고정 우선순위).

### 3.3 추가 시 최종 카테고리

- **`resolveCategoryForNewTask(rawText)`**  
  - 자동 분류 **꺼짐**: `#task-category` 값만 사용.  
  - 자동 분류 **켜짐**: `detectCategoryFromKeywords(trimmed)` 결과가 있으면 그 값, 없으면 드롭다운 값.

폼 `submit`에서는 위 함수로 구한 카테고리를 **`addTask(..., cat, false)`**에 넘긴다. 중복 확인 다이얼로그 경로도 동일한 `cat`이 유지된다.

### 3.4 적용 범위

- **신규 추가**에만 적용한다. 목록 **인라인 편집** 시 카테고리는 사용자가 선택한 값만 반영되며, 키워드 재분류는 하지 않는다.

## 6. 접근성·UX

- 힌트 영역에 **`aria-live="polite"`**를 두어, 자동 분류가 켜진 상태에서 추천·비매칭 안내가 보조 기술로 전달될 수 있게 하였다.
- 체크박스는 **`<label>`로 감싼** 표준 패턴으로 클릭 영역과 이름을 연결하였다.
- 자동 분류가 꺼지면 힌트를 **`hidden`** 처리하여 불필요한 중복 안내를 줄인다.

## 8. 한계·향후 확장 아이디어

- 키워드는 **코드 상수**이며 사용자 정의 UI는 없다. 필요 시 `localStorage`나 가져오기 패키지에 **사용자 규칙**을 넣는 확장을 고려할 수 있다.
- **부분 문자열** 매칭이므로, 의도하지 않은 단어가 다른 단어의 일부로 포함되면 오분류가 날 수 있다. 길이 우선 정렬로 위험을 완화하였다.
- 다국어 키워드 확장 시 `CATEGORY_KEYWORDS`만 보강하면 된다.

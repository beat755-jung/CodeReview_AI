# My Tasks — 할 일 관리 앱 프로젝트 보고서

**작성일:** 2026년 4월 30일  
**프로젝트 경로:** `c:\DEV\ToDo_list`  
**구성:** 순수 HTML / CSS / JavaScript (빌드 도구 없음)

---

## 1. 개요

브라우저에서 동작하는 단일 페이지 할 일 관리 앱입니다. 입력·목록·완료 표시·삭제를 지원하며, `localStorage`로 새로고침 후에도 데이터가 유지됩니다.

---

## 2. 파일 구성

| 파일 | 역할 |
|------|------|
| `index.html` | 문서 구조, 스크립트·스타일 연결 |
| `style.css` | 레이아웃·색·호버·트랜지션 |
| `script.js` | 상태 관리, DOM 렌더링, 저장소 입출력 |

---

## 3. HTML 구조 요약

- **제목:** `h1.app__title` — 표시 문구 "My Tasks"
- **입력 영역:** `form#task-form` — 텍스트 `input#task-input`, `type="submit"` 추가 버튼
- **목록:** `ul#task-list` — 스크립트에서 항목 동적 생성, `aria-live="polite"`로 목록 변경 시 보조 기술에 알림
- 접근성: 입력 필드에 시각적으로 숨긴 `label` 연결

---

## 4. 기능 명세

| 기능 | 설명 |
|------|------|
| 할 일 추가 | 폼 제출(추가 버튼 클릭 또는 입력란에서 Enter). 공백만 있는 문자열은 무시 |
| 할 일 삭제 | 각 행의 `×` 버튼 |
| 완료 토글 | 체크박스로 완료/미완료 전환 |
| 완료 표시 | 완료 시 텍스트에 `task-item__text--done` 클래스 적용 → 취소선·톤 다운 |
| 빈 목록 | 항목이 없을 때 안내 문구 표시 |
| 정렬 | `createdAt` 기준 오름차순(생성 순) |

---

## 5. 데이터 모델 및 저장

### 5.1 항목 스키마

각 할 일 객체는 아래 필드를 가집니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | string | 고유 식별자 (`crypto.randomUUID` 우선, 미지원 시 대체 문자열) |
| `text` | string | 할 일 내용 |
| `completed` | boolean | 완료 여부 |
| `createdAt` | string | 생성 시각 ISO 8601 (`Date.prototype.toISOString`) |

### 5.2 localStorage

- **키:** `myTasks:v1`
- **값:** 위 객체의 배열을 `JSON.stringify`한 문자열
- **저장 시점:** 추가(`addTask`), 삭제(`removeTask`), 완료 토글(`toggleComplete`) 직후 `saveTasks()` 호출
- **로드:** 초기화 시 `loadTasks()` — JSON 파싱 실패 또는 형식 불일치 항목은 제외하고 안전하게 복원

---

## 6. 스타일링 요약 (CSS)

- **레이아웃:** 카드형 흰 배경, 그림자, 둥근 모서리(`--radius`)
- **폭:** `.app`에 `max-width: 600px`, 좌우 `margin: auto`로 중앙 정렬
- **배경:** 그라데이션 페이지 배경
- **CSS 변수:** 색·그림자·트랜지션 시간 등 `:root`에 집중 관리
- **인터랙션:** 폼 포커스, 버튼·삭제 버튼·목록 행에 호버 및 `transition` 적용

---

## 7. 실행 방법

1. 파일 탐색기에서 `index.html`을 더블클릭하거나, 브라우저에서 해당 파일을 연다.
2. 동일 출처 정책상 `file://`로 열어도 `localStorage`는 일반적으로 동작한다. 배포 시에는 웹 서버에 올려 `http(s)://`로 제공하는 것이 권장된다.

---

## 8. 기술적 참고 사항

- 스크립트는 IIFE로 전역 오염을 최소화하고 `"use strict"`를 사용한다.
- DOM은 `render()`에서 전체 목록을 다시 그리는 단순 패턴이다. 항목 수가 많아지면 가상 리스트 등으로 최적화할 여지가 있다.
- 향후 확장 예시: 편집, 필터(전체/완료/미완료),보내기/가져오기, 다국어.

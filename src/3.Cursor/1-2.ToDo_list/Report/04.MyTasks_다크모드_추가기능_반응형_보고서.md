## 1. 개요

우측 상단 **테마 토글**, **다크 모드 색상 체계**, **localStorage 영속화**, **색·배경 전환 애니메이션**을 적용하였다. 이와 함께 **완료 항목 일괄 삭제(확인 다이얼로그)**, **실시간 검색**, **남은 할 일 배지**, **빈 목록 문구**, **키보드 단축키**, **모바일(480px 이하) 레이아웃·터치 최소 크기**, **토스트 피드백**을 추가하였다.

## 2. 다크 모드

### 2.1 UI·동작

| 항목 | 설명 |
|------|------|
| 위치 | `header` 내 `.app__header-top` — 제목(`h1.app__title`)과 같은 행 **우측** |
| 컨트롤 | `#theme-toggle` (시각적으로 숨긴 체크박스) + `label.theme-switch__track` (슬라이드 트랙·썸·아이콘) |
| 체크 상태 | 체크 시 다크, 해제 시 라이트 (`applyTheme`) |
| 초기 깜빡임 완화 | `head` 내 인라인 스크립트에서 `localStorage` 키를 읽어 `document.documentElement`에 `data-theme="dark"` 선적용 후 CSS 로드 |

### 2.2 색상·토큰

- 라이트/다크 공통으로 **CSS 변수**(`:root`, `html[data-theme="dark"]`)로 배경·카드·본문·보조색·테두리·입력 배경 등을 전환한다.
- 다크 모드 요구 색상: 배경 `#1A1A1A`, 카드 `#2D2D2D`, 본문 `#E0E0E0` (`--bg`, `--card`, `--text`에 매핑). 그라데이션 끝색 등은 동일 톤에 맞춘 보조 변수를 사용한다.
- **전환:** `--theme-transition`(0.28s)을 `body`, 카드형 블록, 입력, 테마 스위치 썸 등에 부여하여 테마 전환 시 **부드럽게** 바뀌도록 한다.
- `html`에 `color-scheme: light` / `dark`를 테마에 맞게 지정한다.

### 2.3 저장소

| 키 | 값 | 용도 |
|----|-----|------|
| `myTasks:theme:v1` | `"light"` 또는 `"dark"` | 사용자 테마 선택 유지 |

## 3. 추가 기능

### 3.1 완료된 항목 모두 삭제

- 버튼: `#btn-clear-completed` (문구: 「완료된 항목 모두 삭제」).
- **전역** 기준으로 `completed === true`인 항목을 모두 제거한다(현재 카테고리 필터와 무관).
- 완료 항목이 없으면 버튼 **비활성화**(`disabled`, `aria-disabled`).
- 확인: `#dialog-clear-completed` (`<dialog>`) + 내부 `form[method="dialog"]`. 삭제 시 `returnValue === "ok"`일 때만 배열에서 제거·저장·`render()`. `<dialog>` 미지원 환경에서는 `window.confirm`으로 대체한다.

### 3.2 할 일 검색

- 입력: `#task-search`, `type="search"`, `input` 이벤트로 **실시간** 반영.
- `searchQuery` 문자열을 기준으로 **현재 카테고리 필터에 해당하는 목록**에서 `text`를 소문자 비교·부분 일치 필터한다(trim 후 소문자화).
- 검색으로 표시할 항목이 없으면 빈 행 메시지: **「검색 결과가 없습니다.」**

### 3.3 남은 할 일 배지

- 요소: `#remaining-badge` (툴바 내, `aria-live="polite"`).
- **현재 카테고리 필터** 기준 **미완료** 개수와 필터 라벨(예: `남은 할 일 3개 · 업무`)을 표시한다.
- 미완료 개수가 **변경될 때만** 짧은 시각 피드백(`.remaining-badge--pulse`)을 준다.

### 3.4 빈 상태 메시지

| 조건 | 메시지 |
|------|--------|
| 저장된 할 일이 **0건** | 「할 일이 없습니다. 추가해보세요!」 |
| 필터 결과 **0건**(할 일은 있으나 해당 카테고리 없음) | 「이 카테고리에 표시할 할 일이 없습니다.」 |
| 검색 결과 **0건** | 「검색 결과가 없습니다.」 |

### 3.5 사용자 피드백(토스트)

- 컨테이너: `#toast-region` (`aria-live="polite"`, `aria-atomic="true"`).
- 예시: 할 일 추가·수정·단건 삭제·완료 일괄 삭제·테마 전환 시 짧은 문구 표시 후 수 초 뒤 제거.
- 시각 구분: 기본·`toast--success`·`toast--info` 클래스.

## 4. 키보드 단축키

문서 캡처 단계에서 `capture: true`로 `keydown`을 등록한다. `Alt` 조합이며 `defaultPrevented`가 이미 설정된 경우 무시한다.

| 단축키 | 동작 |
|--------|------|
| **Alt+N** | `#task-input`에 포커스 |
| **Alt+1** | 필터 **전체** |
| **Alt+2** | 필터 **업무** |
| **Alt+3** | 필터 **개인** |
| **Alt+4** | 필터 **공부** |
| **Alt+D** | 다크/라이트 **토글** (`toggleTheme` — 저장·토스트 포함) |

필터 전환 시 기존과 동일하게 `editingTaskId`를 초기화한 뒤 `render()`한다.

## 5. 반응형·터치

### 5.1 뷰포트

- **`max-width: 480px`**: 앱 여백·헤더 패딩 축소, 제목 폰트 소폭 조정, 툴바 세로 스택, 카테고리 버튼 2열 그리드, 폼 행 세로 배치·제출/셀렉트 전체 폭, 일괄 삭제 버튼 전체 폭, 확인 다이얼로그 버튼 세로·역순 배치 등.

### 5.2 터치 목표

- CSS 변수 **`--touch-min: 44px`** 를 필터 버튼, 폼 입력/셀렉트/제출, 목록 삭제 버튼, 다이얼로그 버튼, 검색 입력 등에 반영하여 **최소 터치 영역**을 확보한다.

## 8. 확인 권장 사항

1. 라이트·다크 전환 후 **새로고침**해도 테마가 유지되는지 본다.  
2. 검색어 입력·삭제 시 목록이 **지연 없이** 바뀌는지, 세 가지 빈 상태 문구가 각각 나오는지 확인한다.  
3. 완료 항목을 둔 뒤 **일괄 삭제** → 취소/확인 각각 동작과 토스트를 확인한다.  
4. **Alt+N / Alt+1~4 / Alt+D**를 브라우저·OS에 따라 동작이 겹치지 않는지 확인한다(브라우저 기본 메뉴와 충돌 시 OS/브라우저 설정 참고).  
5. 폭 480px 이하(개발자 도구 반응형)에서 버튼·입력 **탭/터치**가 잡히는지 본다.  
6. OS **애니메이션 줄이기** 설정 시 `prefers-reduced-motion`에 따라 전환·토스트·배지 펄스가 완화되는지 확인한다.

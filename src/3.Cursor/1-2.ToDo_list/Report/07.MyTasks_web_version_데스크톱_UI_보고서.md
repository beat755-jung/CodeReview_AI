## 1. 개요

루트의 `index.html`은 **최대 너비 600px** 중앙 정렬로 모바일·좁은 창에 맞춰져 있다. **와이드스크린·전체 화면**에서도 정보 밀도와 가독성을 살리기 위해, 동일 기능을 유지한 채 **2열 레이아웃(좌측 패널 + 우측 메인)** 으로 재배치한 변형을 **`web_version`** 폴더에 두었다. 동작 로직은 루트의 `script.js`와 동일한 복사본을 사용하며, **localStorage 키**도 같아서 같은 출처에서 열면 기존 할 일 데이터가 그대로 이어진다.

## 2. 폴더 및 파일 구성

| 경로 | 설명 |
|------|------|
| `web_version/index.html` | `<main class="app app--desktop">` 및 **메인·사이드바** 래퍼 구조. 문서 제목은 `My Tasks (웹)`으로 구분 |
| `web_version/style.css` | 프로젝트 루트 `style.css`를 복사한 뒤, **데스크톱·좁은 창**용 보조 규칙을 하단에 추가 |

루트의 `index.html` / `style.css` / `script.js`는 **수정하지 않았다.**

## 3. HTML 구조

### 3.1 역할 분담

| 영역 | 클래스 | 포함 블록 |
|------|--------|-----------|
| 메인 열 | `.app__main` | `header`(제목·테마), `.app__toolbar`(배지·검색), `#task-form`, `.list-actions`, `#task-list`, `#manual-sort-hint` |
| 사이드바 | `.app__sidebar` (`aside`, `aria-label`로 영역 설명) | `#dashboard`, `#data-toolbar`, `#sort-toolbar`, `#task-filters` |

### 3.2 DOM 순서와 시각 순서

**DOM 순서**는 **메인 → 사이드바**이다. 좁은 화면(`max-width: 960px`)에서는 `display: block`으로 세로 쌓이므로, **헤더·입력·목록이 먼저** 나오고 **대시보드·JSON·정렬·필터**가 아래에 이어진다.

**960px 초과**에서는 CSS Grid로 **시각적으로만** 열을 바꾼다. `.app__sidebar`에 `grid-column: 1`, `.app__main`에 `grid-column: 2`를 지정하여 **왼쪽에 패널, 오른쪽에 할 일 작업 영역**이 되도록 했다.

## 4. 스타일(레이아웃) 요약

### 4.1 데스크톱 (`min-width: 961px`)

| 항목 | 내용 |
|------|------|
| 컨테이너 | `.app.app--desktop`: `max-width: min(1680px, 100%)`, `min-height: 100vh`, 가로 패딩 `clamp(1rem, 2vw, 2rem)` |
| 그리드 | `grid-template-columns: minmax(260px, 22rem) minmax(0, 1fr)` — 좌측 고정 폭대, 우측 가변 |
| 사이드바 | `position: sticky`, `top: 1rem`, `max-height: calc(100vh - 2rem)`, 세로 스크롤(`overflow-y: auto`)로 긴 패널 대응 |
| 메인 열 | `flex` 세로 스택, `gap: 1rem`, `min-width: 0`(그리드 자식 오버플로 방지) |
| 할 일 목록 | `.app__main .task-list`에 `flex: 1 1 auto`, `min-height: min(55vh, 560px)`로 **세로 공간 활용** |
| 패널 내부 | 사이드바의 `데이터`·`필터`·`JSON` 버튼 등은 **세로 스택**(버튼 전폭)으로 정돈 |
| 토스트 | `.toast-region` 최대 너비 `min(90vw, 520px)`로 와이드 화면에서도 과도한 가로 길이 제한 |

데스크톱에서 사이드바·메인의 **직계 자식**에 대해 `margin-bottom: 0`으로 두고, 열 내부 간격은 **`gap`**으로 통일했다.

### 4.2 좁은 창 (`max-width: 960px`)

| 항목 | 내용 |
|------|------|
| 레이아웃 | `.app.app--desktop`을 `display: block`, `max-width: 600px`, 중앙 `margin` — 루트 앱과 유사한 **한 열** |
| 그리드 해제 | `.app__main`, `.app__sidebar`의 `grid-column` / `grid-row`를 `auto`로 되돌림 |
| 사이드바 | `position: static`, 높이 제한 해제, 상단 여백 `margin-top: 1rem` |

루트 `style.css`에 있는 **`max-width: 480px`** 미디어쿼리(폼 세로 배치 등)는 그대로 적용되며, `web_version` 전용 규칙과 **함께** 동작한다.

## 5. 스크립트·데이터

- `web_version/script.js`는 **요소 ID**(`#task-list`, `#dashboard`, `#sort-toolbar` 등)로만 DOM에 접근하므로, **HTML에서 블록 위치만 바뀐 경우**에도 동작한다.
- 저장소 키(`myTasks:v2`, 필터·정렬·테마·자동분류 등)는 루트와 **동일**하다. 같은 브라우저·같은 출처(예: 로컬에서 동일 폴더를 `file://`로 연 경우 동일 경로)라면 **한쪽에서 저장한 목록이 다른 쪽에도 반영**될 수 있으니, 배포 시 출처·경로를 구분할 계획이 있다면 키 분리를 검토하면 된다.

- `web_version` 전용 **저장소 키 접미사**로 루트 앱과 데이터를 완전히 분리하기.

## 1. 개요

본 문서는 **JSON 데이터 보내기/가져오기(백업 확인)**, **정렬 옵션·수동 드래그 정렬·정렬 상태 저장**, **검색·저장 디바운스와 목록 렌더링 최적화**, **접근성(ARIA·포커스·스크린 리더)**, **중복 경고·삭제 Undo·오늘의 격언·완료율 응원 문구**, **엣지 케이스·에러 처리**를 반영한 **완성 단계** 구현을 정리한다.

## 2. 데이터 보내기·가져오기

### 2.1 JSON 보내기

| 항목 | 설명 |
|------|------|
| UI | `#btn-export-json` — 레이블 「JSON보내기」, `aria-describedby`로 숨김 설명(`#export-hint`) 연결 |
| 페이로드 | `version`, `app`, `exportedAt`, `sortMode`, `tasks`(각 항목 `id`, `text`, `completed`, `createdAt`, `category`, `sortIndex`) |
| 파일명 | `mytasks-export-{ISO 타임스탬프}.json` 형태 |
| 실패 시 | Blob/다운로드 불가 시 토스트(`error`, assertive) |

### 2.2 JSON 가져오기

| 항목 | 설명 |
|------|------|
| UI | `#input-import-json` — `accept=".json,application/json"`, 라벨 「JSON 가져오기」 |
| 형식 | 순수 **배열** 또는 `{ tasks: [...], sortMode?: ... }` 래퍼 모두 허용 |
| 검증 | 빈 파일, JSON 파싱 오류, 지원하지 않는 루트 형식, 유효 행 없음, **잘못된 `createdAt`** 행 제외 |
| 용량 | 약 **5MB(문자 수)** 상한 초과 시 거부 |
| ID 충돌 | 가져오기 배치 내 중복 `id`는 새 `id` 발급 |

### 2.3 가져오기 전 백업 확인

- 다이얼로그: `#dialog-import-backup` (`<dialog>` + `form[method="dialog"]`).
- 선택지: **취소** / **백업 없이 가져오기** / **백업 후 가져오기**.
- 「백업 후」: 현재 목록을보내기와 동일한 구조로 **파일 다운로드**한 뒤, 가져온 데이터로 **목록 교체**.
- 닫힌 뒤 `#input-import-json` 등 **`focusReturnEl`**로 포커스 복귀.

## 3. 정렬·수동 순서(DnD)

### 3.1 정렬 모드

| 모드 | 동작 요약 |
|------|-----------|
| 생성일 | `createdAt` 오름차순(동률 시 완료 여부·`id`로 안정 정렬) |
| 카테고리 | `work` → `personal` → `study` 순, 동률 시 생성일 |
| 완료 여부 | 미완료 우선, 동률 시 생성일 |
| 수동(드래그) | `sortIndex` 오름차순, `ensureSortIndices()`로 정규화 |

- UI: `#sort-toolbar` — `role="radiogroup"`, `aria-labelledby="sort-toolbar-label"`, 각 버튼 `role="radio"`, `aria-checked` 동기화.
- 저장 키: **`myTasks:sortMode:v1`**.

### 3.2 드래그 앤 드롭·키보드

| 항목 | 설명 |
|------|------|
| 조건 | **수동** 모드 + **전체** 필터 + **검색어 비움** + 인라인 편집 아님 |
| 핸들 | `.task-item__drag`(⋮⋮)에만 `draggable="true"` — 체크박스와 드래그 충돌 방지 |
| 하이라이트 | `dragover` 시 **전체 `render()` 호출 없이** 클래스만 갱신하여 드래그 세션 유지 |
| 키보드 | 핸들 포커스 시 **Alt+↑/↓**로 인접 항목과 순서 교환(스크린리더·키보드 대안) |
| 안내 | 조건 불충족 시 `#manual-sort-hint`에 안내 문구 표시 |

### 3.3 데이터 필드

- 각 할 일에 선택 필드 **`sortIndex`**(수동·일관 순서). 기존 `myTasks:v2` 배열과 호환되도록 로드 시 보정한다.

## 4. 성능

| 기법 | 설명 |
|------|------|
| 저장 디바운스 | `saveTasks()` — 짧은 간격의 연속 변경을 **한 번의 `localStorage` 쓰기**로 묶음 |
| `beforeunload` | **`flushSaveTasks()`**로 종료 직전 미반영분 저장 |
| 검색 디바운스 | 입력 후 **약 220ms** 뒤 `searchQuery` 반영 + `requestAnimationFrame` 기반 `scheduleRender()` |
| DOM | 목록 행을 **`DocumentFragment`**에 모아 한 번에 `#task-list`에 부착 |

100건 이상에서도 불필요한 저장·렌더 호출을 줄이는 데 초점을 두었다(가상 리스트는 범위에 포함하지 않음).

## 5. 접근성

| 영역 | 조치 |
|------|------|
| 영역·그룹 | 데이터 도구 `role="region"`, 정렬 `radiogroup` + `aria-labelledby` |
| 목록 | `#task-list`에 `aria-label="할 일 목록"`, 각 `li`에 `role="listitem"` 및 **요약 `aria-label`**(완료·카테고리·본문·상대 시각) |
| 컨트롤 | 검색·추가·삭제·완료 일괄 삭제·테마 등에 **`aria-label`** 또는 기존 숨김 레이블 유지 |
| 알림 | 일반 토스트: `#toast-region` (`polite`). 중요 오류/가져오기 결과: **`#toast-region-assertive`** + `role="alert"` |
| 다이얼로그 | `close` 후 **`focusReturnEl.focus()`**로 트리거 요소로 포커스 복귀 |
| Undo | `#undo-bar` — `role="status"`, `aria-live="polite"` |

## 6. 추가 UX·동작

### 6.1 중복 확인

- 추가·수정 시 문구를 **공백 정규화·소문자 비교**로 동일 여부 판별.
- 중복이면 `#dialog-duplicate` — 「취소」/「그래도 추가」 또는 편집 시 「그래도 수정」.
- 편집 중 취소 시 **`render()`**로 편집 UI 유지.

### 6.2 삭제 Undo

- 단일 삭제 완료 후 **`undoState`**(복구할 항목·삽입 인덱스)·타이머(약 8초).
- `#undo-bar`에서 「실행 취소」 — 이미 동일 `id`가 있으면 안내 후 중단.
- **가져오기**, **완료 일괄 삭제** 시 Undo 바 **숨김**으로 상태 불일치 방지.

### 6.3 오늘의 격언·완료율 응원

- 격언: **`sessionStorage`** 키 `myTasks:quoteDay:v1`에 **날짜+문구** 저장 — 같은 날·같은 탭에서는 동일 문구.
- 응원: 대시보드 완료 **퍼센트 구간**별 문구(`#completion-cheer`, `aria-live="polite"`).

## 7. 엣지 케이스·에러 처리

| 상황 | 처리 |
|------|------|
| `localStorage` 할당 실패 | `QuotaExceededError` 등 구분 메시지, 필터/정렬 저장 실패 시 토스트 |
| 가져오기 실패 | `FileReader` 오류, 파싱·스키마·용량·유효 행 없음 각각 사용자 메시지 |
| `<dialog>` 미지원 | 완료 일괄 삭제·중복·가져오기 백업에 **`window.confirm`** 대체 경로 |
| 드래그 중 DOM 파괴 | `dragover`에서 전체 리렌더 금지 |
| 테마·필터 | 기존 키(`myTasks:theme:v1`, `myTasks:filter:v1`)와 동작 유지 |
